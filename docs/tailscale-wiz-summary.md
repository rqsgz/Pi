# 使用 Tailscale 控制 WiZ 灯泡 — 简要总结

## 核心原理
Tailscale 基于 WireGuard 建立加密的 overlay/VPN，把物理上不同网络的设备连成同一逻辑网络（每台设备分配 100.x.x.x 虚拟 IP）。控制流量优先点对点穿透，穿透失败时通过加密中继（DERP）。

## 两种控制方式
- 厂商云：树莓派 → Internet → WiZ 云 → 灯泡（依赖云服务，配置简单）。
- 局域网本地控制：树莓派直接访问灯泡的局域网 IP（即时、可控），要求控制器能到达灯泡所在局域网。

## Tailscale 如何实现跨网本地控制
- 在灯泡所在局域网运行一个 Tailscale 节点并广告子网路由，例如：
  tailscale up --advertise-routes=192.168.1.0/24
- 在 Tailscale 管理控制台接受该子网路由后，远端设备可通过该路由访问局域网内的私有 IP。
- 需要在广告子网的设备上启用 IP 转发，并配置相应防火墙/转发规则。

## 局域网发现与限制
- mDNS / 广播发现通常不会自动穿越子网路由。
- 解决方法：使用静态 IP、在本地运行代理（转发控制请求）、或部署 mDNS 转发器；或改用厂商云 API。

## 安全与注意事项
- 子网路由会把该局域网暴露给有权限的 Tailscale 设备，务必仅授予可信账号/设备权限。
- 启用路由时配置防火墙以限制非必要流量，注意 DERP 会增加延迟但数据仍被加密。

## 常见排查步骤
- 在两端运行 `tailscale status`，确认节点与路由是否可见。
- 在广告子网设备上确认 IP 转发（Linux: `sysctl net.ipv4.ip_forward=1`）与防火墙规则。
- 若能连到广告节点但无法发现设备，优先尝试静态 IP 或本地代理方案。
